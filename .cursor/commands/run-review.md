# コードレビュー（差分ベース）

現在のブランチの変更をレビューする。マージ先は **`main`** を既定とする（異なる場合は置き換える）。

## 手順

1. 差分を取得する  

   ```bash
git fetch origin
git diff main...HEAD
```

2. レビュー観点・出力形式は **`.cursor/agents/reviewer.md`** に従う（Critical / Suggestion / Nice to have）。
3. 変更されたファイルを必要に応じて読み、指摘はファイル名・行番号・修正案まで具体化する。
4. サブエージェントを使う場合は、`reviewer` エージェントの定義（`.cursor/agents/reviewer.md`）をプロンプトに含めて依頼する。

## 完了条件

- 重大な指摘（Critical）がある場合: 修正案を出し、実装側の修正後に再度本手順を実行する
- 問題なければレビュー結果で「マージ可能」と判断できる状態にする

## 注意

- セキュリティ・テスト不足・プロジェクト規約（`.cursor/rules/*.mdc`）違反を優先的に挙げる
