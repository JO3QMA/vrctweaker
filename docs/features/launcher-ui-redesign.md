# Launcher（起動プロファイル画面 UI リデザイン）

## 概要

`frontend/src/views/LauncherView.vue` のレイアウト、UX、情報設計を改善する。
起動引数の編集と保存を主目的とし、VRChat の起動そのものはダッシュボードに委ねる。

現状実装は機能的に揃っているが、左右分割が同格に見え、フォームが縦長で、未保存の変更が静かに失われるリスクがある。
本ドキュメントは grill-me セッション（2025-06）で合意した方針の正本とする。

関連コード:

| 層 | ファイル |
|----|----------|
| UI | `frontend/src/views/LauncherView.vue` |
| Storybook | `frontend/src/views/LauncherView.stories.ts` |
| ドメイン | `internal/domain/launcher/` |
| Use case | `internal/usecase/launcher_usecase.go` |
| 永続化 | `internal/infrastructure/sqlite/launcher_repo.go` |
| Wails | `app.go`, `bindings.go` |

ダッシュボードのクイック起動: `frontend/src/views/DashboardView.vue`（本リデザインでは変更しない）。起動は **Quick launch**（`CONTEXT.md`）。未保存の Launcher 編集は反映されない。

用語の正本: リポジトリルート [`CONTEXT.md`](../../CONTEXT.md)（Launcher 節）

## ゴール

- **起動プロファイルの編集と保存**を画面の主目的に据える
- エディタ中心のレイアウトで、よく触る項目を上面に、上級オプションを一か所に集約する
- 未保存の変更を可視化し、プロファイル切り替え時に確認する
- 起動ボタンは残すがセカンダリとし、保存をプライマリアクションにする

## スコープ外

以下は本リデザインでは扱わない。

- プロファイルの複製、並べ替え
- プロファイルのインポート / エクスポート
- 現状 GUI にない VRChat 起動引数の追加
- ダッシュボード側の起動 UI 変更
- バックエンド API や DB スキーマの変更（UI 層とクライアント状態が中心）
- Default launch profile 削除後の自動昇格（現状どおり、Default が無い状態を許容）

## 画面の役割分担

| 画面 | 役割 |
|------|------|
| **Launcher** | プロファイルの作成、編集、保存、削除。必要ならセカンダリとして起動 |
| **Dashboard** | Default launch profile での Quick launch（主な起動導線。DB 保存済みのみ参照） |
| **Gallery** | World join 時は既定 Launch profile を使用（既存どおり） |

## レイアウト

### 全体構成

エディタを主画面とし、プロファイル一覧は折りたたみ可能な狭いサイドバーに置く。

```text
┌──────────────────────────────────────────────────────────┐
│ [≡] サイドバー（折りたたみ可） │ エディタ（主画面）           │
│  プロファイル一覧              │ [未保存バナー]               │
│  未保存インジケータ ●          │ プロファイル名               │
│                                │ 既定 / デスクトップ / 表示   │
│                                │ カスタム引数                 │
│                                │ ▼ すべてのオプション         │
│                                │    表示・性能                │
│                                │    デバッグ・上級            │
│                                │              [起動] [保存] ⋯ │
└──────────────────────────────────────────────────────────┘
```

### サイドバー

- 幅は現状（240px 固定）より狭く、コンパクトな一覧とする
- 各行: プロファイル名、既定タグ（`isDefault`）、未保存ドット（後述）
- **開閉状態**: `localStorage` などで前回の状態を記憶する
- 新規プロファイル追加ボタンはサイドバー内に維持する

### エディタ上部

- 未保存バナー（後述）
- 右寄せアクション: `[このプロファイルで起動]`（ghost / default）、`[保存]`（primary）
- オーバーフローメニュー `⋯`: 削除（確認ダイアログは現状どおり `ElMessageBox`）

## エディタ上面（常時表示）

以下のみを折りたたみの外に置く。

| 項目 | 対応する起動引数（概念） |
|------|--------------------------|
| プロファイル名 | DB フィールド `name` |
| 既定プロファイル | `isDefault` |
| デスクトップモード | `-no-vr` |
| 表示モード | `-screen-fullscreen` / `-windowed` / `-popupwindow` |
| カスタム引数 | `custom`（自由テキスト） |

解像度、モニター、FPS、優先度、デバッグ系などは **すべてのオプション** に移す。

## すべてのオプション（折りたたみ）

現状の **詳細設定** と **クリエイター・デバッグ** の 2 段 `el-collapse` を、外側 1 つの折りたたみに統合する。
内側は見出し 2 ブロックで区切る（入れ子の `el-collapse` は使わない）。
ブロック見出し（「表示・性能」「デバッグ・上級」）は UI ラベルと i18n のみ。`CONTEXT.md` では **Advanced launch options** までをドメイン用語とする。

### ブロック 1: 表示・性能

現状の「詳細設定」に相当する項目。

- 解像度（プリセット + 手動。`resolution-selection-ui` ルールに従う）
- モニター（`-monitor`）
- FPS（`--fps`）
- レジストリ登録スキップ
- プロセス優先度、メインスレッド優先度
- VRChat 内部プロファイル番号（`--profile`）

### ブロック 2: デバッグ・上級

現状の「クリエイター・デバッグ」に相当する項目。

- デバッグ GUI、SDK ログ、Udon デバッグ
- watch-worlds / watch-avatars
- enforce-world-server-checks
- MIDI、ignore-trackers、video-decoding、AMD stutter、OSC、affinity

### 折りたたみの初期開閉

選択中プロファイルの起動引数をパースした結果、上級オプション（上面に出していない項目）のいずれかが有効なら **開いた状態** で表示する。
すべて無効なら **閉じた状態** とする。

有効判定は `syncValueOptionsEnabled` 相当のロジックを拡張し、ブロック 1・2 の各フィールドを対象とする。

## 保存と未保存の挙動

### 保存方式

- **明示的保存**のみ（自動保存は行わない）
- 保存時は現状どおり `mergeLaunchArgsForGUI` → `saveLaunchProfile` → 一覧再取得

### 未保存の検知

エディタ上の `launchArgs` と `selected`（名前、既定フラグ含む）を、最後に保存または読み込みしたスナップショットと比較する。
差分があれば dirty とみなす。

### 未保存の表示

| 場所 | 内容 |
|------|------|
| サイドバー | 該当プロファイル名の横に未保存ドット（●） |
| エディタ上部 | バナー「未保存の変更があります」+ 保存への導線（保存ボタンと重複してよい） |

### プロファイル切り替え

別プロファイルを選択しようとしたとき、現在のプロファイルが dirty なら確認ダイアログを出す。

- 保存して切り替え
- 破棄して切り替え
- キャンセル（現在の選択のまま）

新規プロファイル追加（`addNew`）時も同様に扱う。対象が **Draft launch profile**（`id` が空）のときも同じ 3 択とし、破棄ならドラフトは消えてから切り替え先を開く。ドラフトはサイドバーに行が無いため、未保存表示はバナーが主となる。

### Launcher 以外への移動

サイドバーで Dashboard や Gallery など **Launcher 以外のルート**へ移動しようとしたとき、現在の Launch profile に Unsaved launch profile edits があれば、プロファイル切り替えと同じ確認ダイアログを出す。

- 保存して移動
- 破棄して移動（Discard launch profile edits）
- キャンセル（Launcher に留まる）

実装は Vue Router の `onBeforeRouteLeave` または同等のナビゲーションガードとする。

### 起動と保存の関係

「このプロファイルで起動」は **保存前の編集中状態** で起動してよい（現状どおり）。
起動はセカンダリアクションであり、保存を強制しない。

## アクション配置

| 操作 | 位置 | スタイル |
|------|------|----------|
| 保存 | エディタ上部右寄せ | `type="primary"` |
| 起動 | 保存の左 | ghost / default（セカンダリ） |
| 削除 | `⋯` メニュー内 | 確認後 `deleteLaunchProfile` |

フォーム最下部の `editor-actions` は廃止し、上部ツールバーに集約する。

## 実装フェーズ

段階的にマージし、各フェーズでテストを通す。

### フェーズ 1: 未保存管理

- dirty 追跡（スナップショット比較）
- プロファイル切り替え時の確認ダイアログ
- Launcher 以外へのルート離脱時の確認ダイアログ（`onBeforeRouteLeave`）
- 未保存バナー、サイドバーのドット
- Vitest: `LauncherView.spec.ts` に未保存系ケースを追加

### フェーズ 2: レイアウト

- 折りたたみサイドバー（開閉状態の localStorage 永続化）
- エディタ中心化（スタイル調整）
- 上部アクションバー（保存 primary、起動 secondary）
- 削除を `⋯` メニューへ移動
- Storybook: `LauncherView.stories.ts` を新レイアウトに合わせて更新

### フェーズ 3: フォーム整理

- 上面フィールドの絞り込み
- 「すべてのオプション」への統合と見出し 2 ブロック
- 折りたたみ初期開閉ロジック
- i18n: `launcher.allOptions` など必要キーを `write-locale-json.mjs` に追加

## 受け入れ条件（DoD）

### 全体

- [ ] 編集・保存が画面の主目的として視覚的に明確である
- [ ] スコープ外項目（複製、インポート、ダッシュボード変更など）に手を入れていない

### 未保存（フェーズ 1）

- [ ] フィールド変更後、保存前にサイドバーにドットが表示される
- [ ] エディタ上部に未保存バナーが表示される
- [ ] dirty 状態で別プロファイルを選ぶと確認ダイアログが出る
- [ ] dirty 状態で Launcher 以外へナビゲートすると確認ダイアログが出る
- [ ] 「破棄して切り替え」で編集内容が失われ、選択だけ変わる
- [ ] 「保存して切り替え」で保存後に切り替わる

### レイアウト（フェーズ 2）

- [ ] サイドバーを折りたためる
- [ ] サイドバーの開閉状態が次回訪問時に復元される
- [ ] 保存が primary、起動が secondary として上部右に配置される
- [ ] 削除は `⋯` からのみ実行できる

### フォーム（フェーズ 3）

- [ ] 上面に名前、既定、デスクトップ、表示モード、カスタム引数のみがある
- [ ] 解像度以下の上級項目は「すべてのオプション」内の 2 ブロックに収まる
- [ ] 上級オプションが有効なプロファイルを開くと折りたたみが開く
- [ ] 上級オプションがすべて無効なら折りたたみは閉じる

### 検証

- [ ] `make fmt`, `make test`, `make lint` が通る
- [ ] `LauncherView` 変更に伴い Storybook ストーリーが更新されている
- [ ] 可能なら `make test-e2e`（`frontend/e2e/navigation.spec.ts` など）
